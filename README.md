# TypeCode — コード × 日本語タイピングゲーム

エンジニア向けのタイピングゲーム。JavaScript・Python・HTML・CSS・SQL のコードと、日本語（ローマ字入力）の両方でタイピング練習できる。バニラJS + Web Audio API のみで作成。

## 特徴

- **コード × 日本語 両対応** — エンジニアが実際に打つ「英語コード＋日本語ローマ字」を一つのゲームで練習
- **ファイルを開くだけで動く** — サーバー不要、`index.html` をダブルクリックするだけ
- **派手な演出** — パーティクル・コンボエフェクト・効果音・シェイク・WPM カラー変化
- **スコア記録** — localStorage にベストスコア・履歴を保存。グラフで推移確認

## 遊び方

1. `index.html` をブラウザで開く（ダブルクリック OK）
2. **Mode・Language・Difficulty・Timer** を選択して「スタート」
3. 画面に表示されたテキストをタイプ
4. 完走するとスコアとWPMが表示される

| キー | 動作 |
|---|---|
| `Tab` | この問題をスキップして次へ |

| `Enter` | スタート画面でゲーム開始 |
| `↑↑↓↓←→←→BA` | イースターエッグ 🎉 |

## モード説明

### Code モード
- JavaScript・Python・HTML・CSS・SQL から言語を選択
- コードを一文字ずつ正確にタイプ
- 大文字・記号・インデントもそのまま入力（英語キーボード想定）

### 日本語モード
- ひらがな・カタカナのテキストを **ローマ字入力** でタイプ
- 画面上部にひらがな、下部にタイプする文字列が表示される
- 例: 「はるのさくら」→ `harunosakura`

### Mix モード
- 日本語（ローマ字）→ コード → 日本語… と交互に切り替わる
- エンジニアの実際の業務入力体験に近い設計
- セグメントインジケーター（🇯🇵 / 💻）で現在の入力種別が分かる

## スコア指標

| 指標 | 説明 |
|---|---|
| **WPM** | Words Per Minute — 1分あたりに打てる単語数（5文字 = 1語換算） |
| **Accuracy** | 正確性 — 全キー入力のうち正解だった割合 |
| **Streak** | 連続正解数（ミスするとリセット） |
| **Max Streak** | ゲーム内の最長連続正解数 |

## 実装のポイント解説

### 日本語モードのローマ字入力

IME（変換）を使わず、**ローマ字文字列をそのまま比較** する設計を採用。  
画面に「タイプすべきローマ字列」を表示し、ユーザーが一文字ずつ打つ。  
日本語は常に `.toLowerCase()` で正規化するため、大文字小文字を気にしなくていい。

```
表示: "はるのさくら"
比較列: "harunosakura"
入力: h → a → r → u → ... (一致したら次へ)
```

### ミックスモードのセグメント管理

各 Mix アイテムは `segments` 配列を持つ。エンジンはセグメントを順に処理し、  
完了時に次セグメントに切り替える。UI は `segIdx` の変化を検出して表示を更新。

```javascript
item.segments = [
  { type: 'ja',   display: 'はいれつをはじめる', romaji: '...' },
  { type: 'code', text: 'const arr = [];' }
]
```

### Web Audio API による効果音合成

外部ライブラリや音声ファイルを一切使わず、`OscillatorNode` と `GainNode` でリアルタイム合成。  
ファンファーレはタイミングをずらして複数音を鳴らす。

```javascript
// クリック音: 高周波の短い矩形波
tone(1100, 0.04, 'square', 0.12)

// ファンファーレ: 4音を100msずつずらして鳴らす
[[523,0],[659,0.12],[784,0.24],[1047,0.36]]
  .forEach(([f, d]) => tone(f, 0.4, 'sine', 0.22, d))
```

### WPM 計算

「5文字 = 1単語」換算の業界標準方式。誤字はカウントせず正解入力のみを使う。

```javascript
wpm = Math.round((correctKeys / 5) / (elapsedSeconds / 60))
```

### デバウンス戦略

| 処理 | 頻度 |
|---|---|
| 文字ハイライト更新 | キー入力ごと（即時） |
| 統計バー更新 | 100ms 間隔の ticker |
| スコア保存 | 完走・タイムアップ時のみ |

## お題の追加方法

`js/words-data.js` の対応カテゴリに項目を追加するだけ。

**コード追加例:**
```javascript
{ id: "js-e-99", text: "const result = await fetch(url);", label: "await fetch" }
```

**日本語追加例:**
```javascript
{ id: "ja-n-e-99", display: "みどりのこうえん", romaji: "midorinokouEn", label: "緑の公園" }
// romaji は小文字で書く（内部で toLowerCase() を適用済み）
```

## ファイル構成

```
typing-game/
├── index.html          HTML構造（style/script埋め込みなし）
├── css/
│   └── style.css       CSS変数でテーマ管理、演出アニメーション
├── js/
│   ├── script.js       ゲームロジック・UI・エフェクト
│   └── words-data.js   お題データ（グローバル変数形式 / file://対応）
├── assets/
│   └── sounds/         （現在未使用 / Web Audio API で代替）
├── README.md
└── LICENSE             MIT
```

> **Note:** お題データを `words-data.json` ではなく `words-data.js` として提供しているのは、  
> `file://` プロトコルで開いた際に `fetch()` が CORS エラーになるため。  
> グローバル変数 `WORDS_DATA` として定義し、`<script>` タグで読み込む方式を採用。
